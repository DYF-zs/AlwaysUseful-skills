mdoc-present

将任意 Markdown 文档转换为自包含的规范化 HTML 演示页。统一使用 warm paper 背景、玻璃态卡片、编辑式排版和语义化徽章系统。

Markdown → HTML 自包含 Glassmorphism Mermaid

核心目标：生成的页面应该像同一设计系统中的文档，而非全新的视觉方向。所有样式通过 inline CSS 自包含，不依赖外部文件。

01

核心 Workflow

  1. 完整读取源 Markdown
  2. 读取 design-tokens.md 获取样式契约
  3. 含产品方案或实施步骤时，读取 layout-patterns.md
  4. 含 Mermaid 代码块时，读取 mermaid-setup.md
  5. 推导文档标题并生成自包含 .html
  6. 预览并验证输出质量
02

不可协商的样式规则

Warm Paper

SVG noise + #faf9f6 → #f0ece4 垂直渐变背景

字体栈

Noto Serif SC 标题 / Noto Sans SC 正文 / JetBrains Mono 代码

Glass 卡片

rgba(255,255,255,.45) 背景 + backdrop-filter blur(24px)

04

接入流程示例

1
准备 Markdown 源文件
确保文档包含一级标题、二级标题和所需表格或图表。
完成标准：文件可正常解析，无语法错误。
2
触发 mdoc-present Skill
在 AI 会话中引用该 skill，提供 Markdown 内容或文件路径。
完成标准：AI 读取并确认文档结构。
3
获取自包含 HTML
AI 生成 inline CSS 的单一 HTML 文件，可直接在浏览器打开。
完成标准：页面样式完整，无外部依赖。
05
