# Canonical Design Tokens and Components

Use this file as the visual contract for every HTML page generated by the `mdoc-present` skill. The style is derived from `分析产出/全域通讯Demo功能需求.html` and should be copied into the generated document as inline CSS.

## Required Reveal Script

```html
<script>
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
      observer.unobserve(entry.target);
    }
  });
}, { threshold: 0.08, rootMargin: '0px 0px -40px 0px' });

document.querySelectorAll('.reveal, .reveal-blur').forEach(el => observer.observe(el));
</script>
```

## Component Notes

- Do not put `overflow: hidden` on `.section`; it can clip the inset left rail.
- Do not use a large left border radius on `.section::before`. The canonical rail is 4px wide and uses `border-radius: 0 4px 4px 0`.
- Keep `.section-num` inside each section even when the source Markdown has its own numbering.
- Hide `.section-num` on very small screens with the provided media rule.
- Use `.note-block` only for page-level context. Use `.note` for inline notes inside sections.
